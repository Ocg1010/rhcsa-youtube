REINICIAR EL PASSWORD DE ROOT
PASO 1: REINICIE EL SERVIDOR E INTERACTUE CON EL GRUB2 PRESIONANDO ALGUNA TECLA ANTES DE QUE TERMINE EL PROCESO DE autoboot, ESPERE A QUE SE RESALTE EN LA ENTRADA DEL KERNEL Y PRESIONE LA LETRA "e" PARA ENTRAR AL MODO EDICIÓN DESPLACESE HACÍA ABAJO PARA ENCONTRAR LA LINEA QUE COMIENZA CON LA PALABRA "linux" Y PRESIONA LA TECLA FIN PARA IR HASTA EL FINAL DE LA LINEA

PASO 2: MODIFIQUE ESTA CADENA DEL KERNEL Y AGREGUE LA PALABRA "rd.break" AL FINAL DE LA LINEA.

PASO 3: PRESIONE "control + x" CUANDO LO HAYA HECHO PARA ARRANCAR DESDE LA shell ESPECIAL Y EL SISTEMA MONTARÁ EL SISTEMA DE ARCHIVOS DE root EN MODO "read only" Y LO MONTARÁ EN EL DIRECTORIO /sysroot. UTILICE EL SIGUIENTE COMANDO PARA EL DIRECTORIO /sysroot APAREZCA COMO MONTADO chroot /sysroot


PASO 4: VUELVA A MONTAR EL SISTEMA DE ARCHIVOS root EN MODO lectura / escritura PARA QUE EL COMANDO passwd SEA CAPAZ DE ASIGNAR UNA CONTRASEÑA Y MODIFICAR EL ARHIVO  /etc/shadow
mount -o remount,rw /

PASO 5: INGRESE EL NUEVO password DE ROOT A TRAVES DEL COMANDO passwd
passwd root

PASO 6: CREAR UN ARCHIVO OCULTO LLAMADO .autorelabel PARA INDICARLE AL SO QUE EJECUTE EL SELinux PARA REETIQUETAR TODOS LOS ARCHIVOS INCLUYENDO EL /etc/shadow QUE FUE ACTUALIZADO CON LA NUEVA CONTRASEÑA DE root EN EL SIGUIENTE REINICIO 
touch .autorelabel

PASO 7: EJECUTE EL COMANDO "exit" PARA SALIR DE LA shell chroot Y DESPUES EJECUTE EL COMANDO reboot PARA REINICIAR EL SISTEMA Y ARRANCAR EL DEFAULT TARGET
exit
reboot
